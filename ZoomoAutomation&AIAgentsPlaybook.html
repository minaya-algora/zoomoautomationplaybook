Step 1: Map Your Process

1. Start Here: Map Your Process

A good, detailed plan is 80% of the work

Before you touch any automation tool, follow this planning process. Map out the process step by step first!

📝 Five Step Mapping Process

Step Question Example
1. 🎯 Define Goal What is the single, clear outcome you want to achieve? "Notify the sales team in Slack about a new B2B lead from our website form."
2. ⚡ Define Trigger What event starts the process? "Form submission on website"
3. 🗺️ Map Steps Write down every single action you do now (manually) from start to finish 1. Lead fills out form → 2. Check if "Budget" > $10k → 3. Get name & company → 4. Format message → 5. Post to Slack
4. 🛠️ List Tools What software do you use now? "Website forms, Slack, maybe Google Sheets"
5. ✅ Define Success How will you know it worked? "Sales team gets notified within 5 minutes of high-value leads"

Pro tip: Sketch first

Draw your process map on paper/whiteboard, take a picture, and upload it to an LLM (like Gemini or ChatGPT) with a prompt like:

"Based on this image of a manual workflow, create a detailed, step-by-step plan for building it as a digital automation."

Step 2: Automation Intelligence

2. Understand Your Automation Needs

What "brain" does your automation require?

Think of automation as three levels on a ladder - from simple rules to intelligent agents.

📊 The 3 Levels of Automation

Level Type How it Works Key Characteristic Example
1 Rule-Based Automation "IF this → THEN that" Follows a fixed path. No thinking involved Copying a new lead from a form to a spreadsheet
2 AI-Enhanced Workflow Traditional automation + AI step: "If This → Think About It (LLM) → Then That" A standard workflow with a "smart" step where an AI analyzes or generates something Classifying an email as "Urgent" or "Not Urgent" before routing it
3 AI Agent Set a goal → figures out how to achieve it Autonomously decides the best steps to achieve a goal. It can use tools and has memory "Handle the next support ticket from start to finish."

Level 1: Rule-Based

"Do exactly this, every single time"

✅ Deterministic: Same input = same output

🛠️ Tools: Google Gems, Zapier, basic Make

Level 2: AI-Enhanced

"Follow my process, but be smart about this one decision"

✅ Smart decision points + flexible input

🛠️ Tools: Zapier + AI, Make + AI, n8n

Level 3: AI Agents

"Here's my goal – figure out the best way to achieve it"

✅ Autonomous + memory + adaptive

🛠️ Tools: n8n agents, custom code

Step 3: Tool Selection

3. Select the Right Tool

How to choose the right tool in 60 seconds

Quick Decision Guide:

  • "Every run should be identical" ➜ Traditional automation (Zapier)
  • "Needs a chatbot/LLM once or twice" ➜ AI-enhanced (Zapier, Make)
  • "Else, or full autonomy" ➜ AI Agent (n8n, custom code)

Golden Rules & Next Steps

4. The Golden Rules

Essential guidelines for success

#1: Be careful with company data

  • We use cloud-based AI models (OpenAI GPT, Google Gemini)
  • Any data you send leaves the Zoomo environment
  • Do NOT send PII or sensitive company data without approval
  • If your project involves sensitive data, consult IT team first

#2: Start simple, then scale

  • Don't over-engineer
  • Build a simple MVP in the most accessible tool first (Zapier/Make)
  • Once you've proven it adds value, consider rebuilding in advanced tools
  • Focus on cost-efficiency and scalability later

AI Glossary

AI Glossary

Key terms explained in plain English

AI Agent

A smart bot you give a goal to. It then figures out the steps and uses other apps ("tools") to achieve that goal on its own.

LLM (Large Language Model)

The "brain" behind AI, like Google's Gemini. It's trained on the internet to understand and write human-like text.

Tool Calling

An AI Agent's ability to use other software. For example, using Google Search to find information or using HubSpot to look up a customer.

RAG (Retrieval-Augmented Generation)

The best way to make an AI smart about Zoomo. We give the AI a library of our own internal documents to read before it answers. Much cheaper than fine-tuning.

API (Application Programming Interface)

a set of rules and specifications that allows different software systems to communicate and interact with each other.

Webhook

A way for an app to send real-time information to another app as soon as an event happens. It's essentially an HTTP callback that triggers an action in another system when a defined event happens in the source system.
